# N-GPT — LLM + RAG сервис для изучения Machine Learning

## Что это

**N-GPT** — это ИИ-ассистент LLM + RAG (Retrieval-Augmented Generation) для получения экспертных ответов на вопросы в области Machine Learning

Проект решает простую, но в то же время актуальную проблему:

> База знаний в сфере ML на русском языке часто сложна для понимания. 
> Я сам столкнулся с этим при изучении даже базовых моделей.
>
> Ключевую роль сыграл наставник, который объяснял сложные вещи простым языком — без перегруженных формул и академических формулировок.
>
> Этот проект — попытка воссоздать такого наставника в виде ИИ:
> система, которая объясняет сложные темы в ML и DL максимально просто — "на пальцах".

N-GPT делает это просто:

> Вы задаете вопрос → N-GPT ищет ответ в специализированной базе знаний по ML/DL → Вы получаете точный и понятный ответ

## Почему не обычные LLM:

Большинство людей, как и я, которые самостоятельно учили языки программирования или ML:

* смотрят лекции
* читают статьи
* гуглят материалы
* используют нейросети для разъяснения

Но потом:

* знание поверхностное — кажется, что понял, но сложно воспроизвести
* перегруженные формулировки из учебников
* каскадная сложность: чтобы понять одну тему, нужно знать несколько других (MSE → logloss → градиентный спуск и т.д.)

**Почему RAG-подход решает это:**

* используется узко-специализированная база знаний по ML
* ответы генерируются на основе релевантного контекста, а не берутся из замуных учебников/баз знаний
* объяснения ориентированы на простоту и понятность

## Как это работает Pipeline: (упрощенно)

1. **Источник данных**

   * тексты (лекции, статьи) которые я тщательно собирал

2. **Preprocessing**

   * очистка текста
   * удаление мусора (таймкоды, шум)
   * нормализация

3. **Chunking**

   * деление текста на смысловые куски (с учетом предложений, регистров и т.п.)

4. **Embeddings по API OpenAI**

   * каждый chunk превращается в вектор + overlap
   * сохраняется в базе

5. **Retrieval**

   * по вопросу ищем релевантные chunks

6. **LLM по API OpenAI**

   * модель получает:

     * вопрос
     * найденные chunks
   * генерирует ответ строго исходя из контекста чанков
